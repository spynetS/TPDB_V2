#203 TP_FAILED_AddRow(TPTable *_self, int _count, ...)

Function to allocate a new table within target Database.
Standard initiation function for "TP_Table" has failed.
Args:
- (TPTable*) Target table.

- (int) How many columns in the table, this is repetitive but it's a good way to keep the function error safe.

- (...{n}) Any data type as long as it fits the structure of the columns defined in the table. {enum TPTable_Column_Types}

							
								// This is one approach of defining a table. There can be other approaches
								TPDatabase *MainDatabase = CreateTPDatabase("MainDatabase", "./db");
	
								AddTable(MainDatabase, "Users");
								SetColumnTypes(MainDatabase->Tables[0], 3, TP_STRING, TP_STRING, TP_INT);
								AddRow(MainDatabase->Tables[0], 3, "Ali", "123", 22);
Notes:
- Make sure you are passing a constant string. ex: "hello".
If you need to pass an allocated string, make sure to free it after

- You must pass the arguments in the same order as the columns are defined type whise.

- If you want to reduce ram usage and lazy load rows on demand. Make sure to set RowsOnDemand in Table to TP_TRUE.

There are no more known reasons or bugs within this function, if it fails, report to the git repository.
